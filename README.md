# 凡人修仙传 · 结构化知识库(fanren KB)

本仓库是网络小说**《凡人修仙传》**(忘语 著,全 2426 章)经 AI 抽取管线提取出的**结构化世界观知识库**——把人物、法宝、功法、势力、地点、境界、事件等从原文中抽出,建成可查询、可追溯、可随时间演化的实体-关系图谱。

> 数据由模型按**章节顺序**逐单元抽取;每个单元抽取完成即 git 提交一次,因此可溯源、抗中断、永不重做已完成的工作。

---

## 这里有什么

- **实体(entities)**:15 类世界观对象,每个实体含别名、属性、跨章节的**时间观测**(随剧情演化的称号/境界/外貌/阵营/存殁等),以及出处文档。
- **关系(relations)**:27 类带方向的关系边(师承、道侣、敌对、持有法宝、所属宗门、冒用身份…),每条边带证据文档与章节序号。
- **文档(docs)**:每个抽取单元(~6 章)一份,含摘要、关键事实、涉及实体、章节范围。
- **来源(sources)**:每个已抽单元对应的原文切片,供溯源核对。

> 抽取进行中:目标覆盖全书 **2426 章 / 405 个单元**。本知识库随抽取持续增长。

## 目录结构

```
taxonomy.json          本体定义:类别 / 实体类型 / 关系类型 / 抽取指引(注入提示词)
kb.json                知识库元信息
docs/<kind>/<slug>/     每单元一目录:knowledge.json(结构化)+ content.md(正文摘要)
entities/<type>/        实体文件(按类型分目录),含别名 / 属性 / observations / 出处
graph/relations.jsonl   全部关系边(三元组 + 证据 + 章节序)
sources/<slug>/         每单元原文切片(溯源)
pending/.state/ledger.jsonl  抽取进度账本(每单元 done/failed + 章节范围)
log.md                  抽取活动日志
indexes/                派生检索索引(.gitignore;可由 docs/entities 重建)
```

`.gitignore` 排除可重生的 `pending/inbox` 切片与派生 `indexes/`,保持仓库精简。

## 本体(taxonomy v2,sequenced)

按【章节】顺序连载,启用 `sequenced` —— 每条事实带章节序号,属性随章节演化(后面变了不算矛盾,系统按序保留历史)。

**3 个类别**:`plot`(剧情) · `character`(人物志) · `lore`(设定/百科)

**15 类实体**:
| 类型 | 含义 | 类型 | 含义 | 类型 | 含义 |
|---|---|---|---|---|---|
| person | 人物 | artifact | 法宝 | technique | 功法 |
| spell | 法术/神通 | beast | 灵兽 | faction | 势力/宗门 |
| place | 地点 | world | 界域 | realm | 境界 |
| item | 物品 | pill | 丹药 | material | 材料 |
| formation | 阵法 | race | 种族 | event | 事件 |

**27 类关系**(节选):师承 / 道侣 / 挚友 / 敌对 / 同门 / 所属宗门 / 持有法宝 / 本命法宝 / 修炼功法 / 境界进阶 / 位于 / 冒用身份(impersonates)… 关系名为 ASCII,属性键/标题/指引可中文。

### 时间观测模型(核心特性)
许多属性会随剧情变化(性别/外貌/称号/道号/性格/五行/小阶段/身份/存殁…)。每次出现都记为一条**带章节序的观测**;展示用的 `attributes` 为派生当前值(标量取最新序、集合取并集),完整历史保留在 `observations` 里 —— 故可查询"第 N 章时某人的境界/称号"。

### 身份借用守卫
主角常借用他人姓名行走(如借"厉飞雨"之名)。此类**冒用之名只生成 `impersonates` 关系边,绝不并入冒名者的别名** —— 避免身份污染与误指(`borrowsIdentity` 机制)。

## 维护方式

- **抽取**:AI 抽取管线,按章节文件名严格升序逐单元处理;密集章节自动二分拆章(章界→段落)以适配模型输出预算。
- **连续性**:每单元抽取时会带入前序章节的"世界状态"(身份转移、死亡等),故**必须按序、不跳过**,否则下游归属易错。
- **版本管理**:每抽完一个单元即 `git commit`(并推送 GitHub),账本 `ledger.jsonl` 记录进度;任何中断都从最近提交恢复,不重做。
- **质量守护**:后台哨兵定期体检并自动清洗(借名污染、自环边等),遇活跃抽取自动避让。
