# 凡人修仙传 · 结构化世界观知识库

本仓库把网络小说**《凡人修仙传》**(忘语 著,全 2426 章)的世界观整理成**结构化数据**:将人物、法宝、功法、势力、地点、境界、事件等从原文中抽出,建成可查询、可溯源、可随剧情演化的**实体-关系图谱**。

---

## 目录结构

```
taxonomy.json                本体定义:类别 / 实体类型 / 关系类型
kb.json                      知识库元信息
docs/<类别>/<单元>/           每个抽取单元一目录
  ├─ knowledge.json            结构化内容:摘要、关键事实、涉及实体、章节范围
  └─ content.md                该单元的正文梳理
entities/<类型>/              实体档案,按类型分目录(每个实体一个 JSON)
graph/relations.jsonl        全部关系边(主语-关系-宾语 + 出处 + 章节序)
sources/<单元>/               每个单元对应的原文切片(供溯源核对)
log.md                       内容变更日志
```

| 路径 | 说明 |
|---|---|
| `taxonomy.json` | 定义这个世界用哪些**类别 / 实体类型 / 关系类型**来描述 |
| `docs/` | 按单元(约 6 章)组织的剧情梳理与该段涉及的实体清单 |
| `entities/` | 每个实体的完整档案:名称、别名、属性、随章节变化的历史、出处文档 |
| `graph/relations.jsonl` | 实体之间的所有关系,每行一条,带证据出处与章节序号 |
| `sources/` | 原著章节切片,任何结论都能回溯到原文 |

---

## 实体(15 类)

每个实体含:**名称 + 别名**、**属性**(可随章节演化)、**出处文档**、以及一份按章节序的**历史观测**。

| 类型 | 含义 | 类型 | 含义 | 类型 | 含义 |
|---|---|---|---|---|---|
| person | 人物 | artifact | 法宝 | technique | 功法 |
| spell | 法术 / 神通 | beast | 灵兽 | faction | 势力 / 宗门 |
| place | 地点 | world | 界域 | realm | 境界 |
| item | 物品 | pill | 丹药 | material | 材料 |
| formation | 阵法 | race | 种族 | event | 事件 |

### 属性会随剧情变化
许多属性不是固定值,而是**随章节演化**的:境界、称号、道号、性格、外貌、五行、阵营、存殁……
每个实体既给出**当前值**(便于直接查看),也保留**完整历史**——因此可以查询"**第 N 章时某人物的境界 / 称号 / 阵营**"。

---

## 关系(27 类)

关系是**有方向**的边:`主语 —关系→ 宾语`,每条都带**证据文档**与**章节序号**(何时成立)。按语义可分为几大类:

| 大类 | 关系示例 |
|---|---|
| 人际 | 师承、徒弟、道侣、挚友、同门、同乡、亲属、结义 |
| 阵营/势力 | 所属宗门、出身门派、隶属、盘踞 |
| 敌对 | 交锋、为敌 |
| 法宝/物品 | 持有、本命法宝、炼制、相配 |
| 功法/修为 | 修炼功法、施展神通、境界进阶 |
| 地缘 | 位于、邻近、故乡 |
| 身份 | 冒用身份(主角借他人之名行走时,记为"冒用"关系,而非真名,身份不混淆) |

---

## 数据特性

- **可溯源**:每个实体属性、每条关系都标注来自原著的**哪一(批)章节**,可回到 `sources/` 核对原文。
- **时间一致**:按章节顺序整理,前期与后期的不同状态都保留为历史,不会互相覆盖或矛盾。
- **多名归一**:同一对象的诸多称呼(本名、外号、道号、敬称…)归并到同一实体;借名/化名则单独标注,不污染真身。
